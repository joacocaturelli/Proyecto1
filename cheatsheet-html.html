¿Qué es HTML?

El Lenguaje de Marcas de Hipertexto (HTML) es el componente más básico de la web y se utiliza para estructurar y presentar contenido. HTML5 no es solo un lenguaje de marcado, sino un conjunto de tecnologías que permite crear aplicaciones web ricas e interactivas con soporte nativo para multimedia y almacenamiento local.

Estructura base (plantilla mínima)

Todo documento HTML5 sigue un esqueleto obligatorio que el navegador necesita para interpretar la página correctamente.

<!doctype html> <!-- Indica que usamos HTML5 -->
<html lang="es"> <!-- Idioma del documento -->
<head>
  <meta charset="utf-8" /> <!-- Codificación -->
  <meta name="viewport" content="width=device-width, initial-scale=1" /> <!-- Responsive -->
  <title>Mi página</title>
</head>
<body>
  <!-- Contenido visible para el usuario -->
</body>
</html>

La importancia del <head>

Esta sección no contiene contenido visible, sino metadatos vitales para el SEO, la responsividad y la conexión con archivos externos.

<title>Título de la pestaña</title>
<meta name="description" content="Resumen para buscadores" />
<link rel="icon" href="/favicon.ico" />
<link rel="stylesheet" href="styles.css" /> <!-- CSS externo -->
<script src="app.js"></script> <!-- JS externo -->

Semántica (Layout típico)

Las etiquetas semánticas describen el significado del contenido, facilitando la accesibilidad para lectores de pantalla y mejorando el SEO.

<header>Contenido introductorio o logo</header>
<nav>Sección de enlaces de navegación</nav>
<main>Contenido único y principal (solo uno por página)</main>
<section>Agrupación temática de contenido</section>
<article>Pieza independiente (post, noticia)</article>
<aside>Contenido relacionado secundario (barras laterales)</aside>
<footer>Pie de página con copyright o contacto</footer>

Jerarquía de Texto

Es fundamental usar los encabezados en orden para definir la importancia del contenido.

<h1>Título principal (máxima importancia)</h1>
<h2>Subtítulo de sección</h2>
<p>Texto normal con <strong>importancia fuerte</strong> y <em>énfasis</em>.</p>
<br /> <!-- Salto de línea -->
<hr /> <!-- Separador temático -->

Hipertexto y Enlaces

Los enlaces son la base de la navegación web, permitiendo conectar documentos y recursos.

<!-- Enlace interno -->
<a href="/contacto">Ir a contacto</a>

<!-- Enlace externo con seguridad -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">Google</a>

Multimedia: Imágenes, Audio y Vídeo

HTML5 permite integrar contenido multimedia de forma nativa sin necesidad de software externo.

<!-- Imagen con descripción -->
<figure>
  <img src="img/foto.jpg" alt="Descripción de la imagen" />
  <figcaption>Pie de foto descriptivo</figcaption>
</figure>

<audio controls src="pista.mp3"></audio>
<video controls width="480" src="demo.mp4"></video>

Listas

Existen listas para elementos sin orden específico y para secuencias de pasos.

<!-- Lista no ordenada -->
<ul>
  <li>Elemento 1</li>
  <li>Elemento 2</li>
</ul>

<!-- Lista ordenada -->
<ol>
  <li>Paso 1</li>
  <li>Paso 2</li>
</ol>

Formularios (Interacción)

Son el puente de comunicación bidireccional entre el usuario y el sitio web.

<form action="/procesar" method="POST">
  <label for="nombre">Nombre:</label>
  <input id="nombre" name="nombre" type="text" required />

  <button type="submit">Enviar</button>
</form>

Tipos de Input y Validación

HTML5 introdujo tipos específicos que mejoran la experiencia de usuario y validan datos automáticamente.

<input type="email" placeholder="email@ejemplo.com" />
<input type="number" min="0" max="100" />
<input type="date" />
<input type="password" /> <!-- Oculta caracteres -->
<input type="checkbox" /> <!-- Selección múltiple -->
<input type="radio" /> <!-- Opción única -->
<input type="file" accept=".pdf" /> <!-- Carga de archivos -->

Atributos Importantes

Proporcionan información adicional y permiten conectar el HTML con CSS y JavaScript.

  • id: Identificador único para el elemento.
  • class: Clase reutilizable para estilos y JS.
  • data-*: Atributos de datos personalizados para usar con JavaScript.
  • aria-label: Ayuda a lectores de pantalla cuando el texto no es suficiente.

SEO Básico y Redes Sociales

El uso de etiquetas Open Graph permite controlar cómo se previsualiza tu sitio en redes sociales.

<meta property="og:title" content="Título para redes" />
<meta property="og:description" content="Descripción del sitio" />
<meta property="og:image" content="https://sitio.com/preview.png" />

Buenas Prácticas de HTML

  • Semántica: Usa siempre etiquetas semánticas en lugar de depender únicamente de <div>.
  • Jerarquía: Mantén el orden correcto de encabezados (h1 → h2 → h3) sin saltar niveles.
  • Accesibilidad (A11y): Incluye siempre el atributo alt en imágenes y asocia label con su input usando id y for.
  • Escritura: Escribe etiquetas y atributos siempre en minúsculas (convención moderna).
  • Cierre: Asegúrate de que todos los elementos se cierran en el orden correcto para evitar fallos de anidamiento.
